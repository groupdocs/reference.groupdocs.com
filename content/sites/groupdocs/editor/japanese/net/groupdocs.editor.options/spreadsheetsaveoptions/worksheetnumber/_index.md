## SpreadsheetSaveOptions.WorksheetNumber property

新しい単一ワークシートのスプレッドシートを作成する代わりに、編集したワークシートを既存のスプレッドシートのコピーに挿入できます (既定の動作)。 0 (デフォルト値) の場合、単一の編集済みワークシートで新しいスプレッドシートが作成されます。入力 EditableDocument インスタンス。このスプレッドシートに挿入されます。

```csharp
public int WorksheetNumber { get; set; }
```

### 備考

ワークシート番号整数プロパティは、既定の状態 (予約値 '0') でない場合、ワークシート番号を表すため、0 からではなく 1 から始まり、その最大値はプレゼンテーション内の既存のすべてのスライドの量です。ただし、指定された値がすべてのスライドの量よりも大きい場合、GroupDocs.Editor はそれを調整して最後のワークシートをマークします。負の値も許可され、ワークシートを最後からカウントします。たとえば、「-1」はスプレッドシートの最後のワークシートを意味し、「-2」は最後から 2 番目のワークシートを意味します。正の値と同様に、負のワークシート番号が指定されたスプレッドシートのワークシートの総数を超えると、次のように調整されます。最初のワークシート. [`InsertAsNewWorksheet`](../insertasnewworksheet)ブール値のプロパティは、これと密接に結合されています.

### 例

与えられたスプレッドシートには 5 つのワークシートがあります: WorksheetNumber = 0; — 指定されたスプレッドシートを無視して、新しいスプレッドシートを作成し、編集したワークシートをそこに入れます。 WorksheetNumber = 1; — 最初のワークシートを edited WorksheetNumber = 2 に置き換えます。 — 2 番目のワークシートを edited WorksheetNumber = 5 に置き換えます。 — 最後の (5 番目の) ワークシートを edit WorksheetNumber = 6 に置き換えます。 — 6 は 5 より大きく、したがってadjusted WorksheetNumber = -1 であるため、最後 (5 番目) のワークシートを編集済みのワークシートに置き換えます。 — "-1" は "最後の既存" を意味するため、最後 (5 番目) のワークシートを編集済みのワークシートに置き換えます WorksheetNumber = -2; — 4 番目のワークシートを edited WorksheetNumber = -3 に置き換えます。 — 3 番目のワークシートを edited WorksheetNumber = -4 に置き換えます。 — 2 番目のワークシートを edited WorksheetNumber = -5 に置き換えます。 — 最初のワークシートを edited WorksheetNumber = -6 に置き換えます。 — 最初のワークシートを編集済みのワークシートに置き換えます。"-6" は 5 より大きいため、adjusted です。
