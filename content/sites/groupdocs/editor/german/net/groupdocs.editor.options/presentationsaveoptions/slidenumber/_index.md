## PresentationSaveOptions.SlideNumber property

Ermöglicht das Einfügen einer bearbeiteten Folie in eine vorhandene Präsentation, anstatt eine neue Präsentation mit einer einzelnen Folie zu erstellen (Standardverhalten). Die Foliennummer ist eine auf 1 basierende Nummer einer Folie in der Präsentation, die in die Editor-Klasse geladen wird. Wenn es 0 ist (Standardwert), wird die neue Präsentation mit einer einzigen bearbeiteten Folie erstellt. Wenn es größer oder kleiner als Null ist und es eine gültige Präsentation gibt, die in die Editor-Klasse geladen ist, wird die bearbeitete Folie, die in der EditableDocument-Eingabeinstanz gespeichert ist, in diese Präsentation eingefügt.

```csharp
public int SlideNumber { get; set; }
```

### Bemerkungen

FoliennummerGanzzahl-Eigenschaft stellt, wenn sie sich nicht im Standardzustand befindet (reservierter Wert „0“), eine Foliennummer dar, beginnt also bei 1 und nicht bei null, und ihr Maximalwert ist die Anzahl aller vorhandenen Folien in einer Präsentation. Wenn der angegebene Wert jedoch größer ist als die Anzahl aller Folien, passt GroupDocs.Editor ihn an, um die letzte Folie zu markieren. Negative Werte sind ebenfalls erlaubt und zählen Folien vom Ende. Zum Beispiel impliziert "-1" die letzte Folie in einer Präsentation, "-2" — die vorletzte usw. Wie bei positiven Werten, wenn die negative Foliennummer die Gesamtzahl der Folien in der gegebenen Präsentation überschreitet, wird sie angepasst die erste Folie. Die[`InsertAsNewSlide`](../insertasnewslide) Die boolesche Eigenschaft ist eng mit dieser gekoppelt.
