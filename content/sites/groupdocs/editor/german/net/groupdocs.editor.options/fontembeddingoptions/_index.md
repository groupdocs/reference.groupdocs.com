## FontEmbeddingOptions enumeration

Optionen zum Einbetten von Schriftarten steuern, welche Schriftartressourcen in das ausgegebene WordProcessing- oder PDF-Dokument eingebettet werden sollen

```csharp
public enum FontEmbeddingOptions : byte
```

### Werte

| Name | Wert | Beschreibung |
| --- | --- | --- |
| NotEmbed | `0` | Betten Sie keine Font-Ressourcen ein, weder von EditableDocument noch vom System. Standardwert. |
| EmbedAll | `1` | Analysieren Sie den Dokumentinhalt aus dem Eingabe-EditableDocument, finden Sie alle verwendeten Schriftarten und betten Sie sie in das Ausgabe-WordProcessing- oder PDF-Dokument ein. An erster Stelle nimmt GroupDocs.Editor Schriftarten aus den Schriftartressourcen in EditableDocument. Wenn sie nicht ausreichen oder fehlen, übernimmt GroupDocs.Editor Schriftarten von OS. |
| EmbedWithoutSystem | `2` | Genau bisEmbedAll , aber schließen Sie die Schriftarten aus, die vom Betriebssystem als Systemfonts behandelt werden |

### Bemerkungen

Optionen zum Einbetten von Schriftarten werden während des Dokumentspeicherns angewendet (vom Zwischen-EditableDocument in das Ausgabe-WordProcessing- oder PDF-Format), diese Aufzählung ist als Eigenschaft in den WordProcessingSaveOptions und PdfSaveOptions enthalten, von wo aus sie verwendet werden sollte
