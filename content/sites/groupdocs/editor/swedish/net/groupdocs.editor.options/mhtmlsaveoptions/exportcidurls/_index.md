## MhtmlSaveOptions.ExportCidUrls property

Anger om CID (Content-ID) URL:er ska användas för att referera till resurser (bilder, teckensnitt, CSS) som ingår i MHTML-dokument. Standardvärdet är`falsk` .

```csharp
public bool ExportCidUrls { get; set; }
```

### Anmärkningar

Som standard refereras till resurser i MHTML-dokument med filnamn (till exempel "image.png"), som matchas mot "Content-Location"-rubriker för MIME-delar. Det här alternativet möjliggör en alternativ metod, där referenser till resursfiler skrivs som CID (Content-ID) URL:er (till exempel "cid:image.png") och matchas mot "Content-ID" headers.

I teorin borde det inte finnas någon skillnad mellan de två refereringsmetoderna och någon av dem borde fungera bra i vilken webbläsare eller e-postagent som helst. I praktiken misslyckas dock vissa agenter med att hämta resurser efter filnamn. Om din webbläsare eller e-postagent vägrar att ladda resurser som ingår i ett MTHML-dokument (inte visar bilder eller inte laddar CSS-stilar), prova att exportera dokumentet med CID URLs.
