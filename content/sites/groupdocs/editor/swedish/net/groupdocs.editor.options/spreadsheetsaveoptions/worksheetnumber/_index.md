## SpreadsheetSaveOptions.WorksheetNumber property

Tillåter att infoga redigerat kalkylblad i en kopia av befintligt kalkylblad istället för att skapa ett nytt kalkylblad för ett kalkylblad (standardbeteende). WorksheetNumber är ett 1-baserat nummer av ett kalkylblad i kalkylbladet, laddat i klassen Editor. Om det är 0 (standardvärde), kommer det nya kalkylarket att skapas med ett enda redigerat kalkylblad. Om det är större eller mindre än noll, och det finns ett giltigt kalkylblad, laddat i klassen Editor, det redigerade kalkylbladet, som representeras av input EditableDocument-instans, kommer att infogas i detta kalkylblad.

```csharp
public int WorksheetNumber { get; set; }
```

### Anmärkningar

Arbetsbladsnummerheltalsegenskapen, om den inte är i standardtillstånd (reserverat värde '0'), representerar ett kalkylbladsnummer, så den börjar från 1, inte från noll, och dess maxvärde är mängden av alla befintliga bilder i en presentation. Men om det angivna värdet är större än antalet av alla bilder, kommer GroupDocs.Editor att justera det för att markera det sista kalkylbladet. Negativa värden är också tillåtna och räknar kalkylblad från slutet. Till exempel betyder "-1" det sista kalkylbladet i ett kalkylblad, "-2" - sist utom ett, etc. Precis som med positiva värden, när negativt kalkylbladsnummer överstiger det totala antalet kalkylblad i det givna kalkylbladet, kommer det att justeras till det första kalkylbladet. Den[`InsertAsNewWorksheet`](../insertasnewworksheet) boolesk egenskap är tätt kopplad till denna.
