## WordProcessingEditOptions.InputControlsClassName property

Tillåter att ange ett klassnamn, som kommer att placeras till "class"-attributen i varje HTML-element, som representerar något fält i det inmatade WordProcessing-dokumentet. Som standard är NULL - "class"-attribut tillämpas inte.

```csharp
public string InputControlsClassName { get; set; }
```

### Anmärkningar

Nästan alla format från WordProcessing-formatfamiljen innehåller fält — specifika dokumentenheter som gör det möjligt att hämta indata från användare. Det finns en mängd olika fält: textrutor, kryssrutor, kombinationsrutor, rullgardinslistor, knappar, datum-/tidsväljare, etc. Alla översätts till de mest lämpliga HTML-strukturerna och -elementen, med bevarande av den angivna användaren data, om de finns i indatadokumentet. I specifika användningsfall krävs det bara att samla in inmatad data på en klientsida istället för att redigera hela dokumentinnehållet. I sådana fall krävs det att man identifierar indatakontroller på något sätt för att hämta dem med deras data på klientsidan. Den här egenskapen gör det möjligt att ange ett klassnamn som kommer att tillämpas för varje inmatningskontroll i HTML-uppmärkning, så klientkod kommer att kunna gå över HTML-dokumentstruktur och samla in data.
