## WordProcessingEditOptions.InputControlsClassName property

Permet de spécifier un nom de classe, qui sera placé dans les attributs 'class' de chaque élément HTML, qui représente un champ dans le document WordProcessing d'entrée. Par défaut est NULL - les attributs 'class' ne sont pas appliqués.

```csharp
public string InputControlsClassName { get; set; }
```

### Remarques

Presque tous les formats de la famille de formats WordProcessing contiennent des champs - des entités de document spécifiques, qui permettent d'obtenir des données d'entrée des utilisateurs. Il existe une grande variété de champs : zones de texte, cases à cocher, zones de liste déroulante, listes déroulantes, boutons, sélecteurs de date/heure, etc. Tous sont traduits dans les structures et les éléments HTML les plus appropriés, tout en préservant l'utilisateur saisi. données, si elles sont présentes dans le document d'entrée. Dans des cas d'utilisation spécifiques, il est uniquement nécessaire de collecter les données saisies côté client au lieu de modifier l'intégralité du contenu du document. Dans ce cas, il est nécessaire d'identifier les contrôles d'entrée d'une manière ou d'une autre pour les récupérer avec leurs données côté client. Cette propriété permet de spécifier un nom de classe, qui sera appliqué pour chaque contrôle d'entrée dans le balisage HTML, afin que le code client puisse traverser la structure du document HTML et collecter des données.
