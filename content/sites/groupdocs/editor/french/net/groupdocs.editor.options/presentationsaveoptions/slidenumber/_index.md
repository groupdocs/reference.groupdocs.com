## PresentationSaveOptions.SlideNumber property

Permet d'insérer une diapositive modifiée dans une présentation existante au lieu de créer une nouvelle présentation à une seule diapositive (comportement par défaut). Le numéro de diapositive est un numéro basé sur 1 d'une diapositive dans la présentation, chargé dans la classe Editor. Si c'est 0 (valeur par défaut), la nouvelle présentation sera créée avec une seule diapositive éditée. S'il est supérieur ou inférieur à zéro et qu'il existe une présentation valide, chargée dans la classe Editor, la diapositive modifiée, stockée dans l'instance d'entrée EditableDocument, sera insérée dans cette présentation.

```csharp
public int SlideNumber { get; set; }
```

### Remarques

DiapositiveNuméroLa propriété entière, si elle n'est pas dans l'état par défaut (valeur réservée '0'), représente un numéro de diapositive, donc elle commence à partir de 1, pas de zéro, et sa valeur maximale est le nombre de toutes les diapositives existantes dans une présentation. Cependant, si la valeur spécifiée est supérieure à la quantité de toutes les diapositives, GroupDocs.Editor l'ajustera pour marquer la dernière diapositive. Les valeurs négatives sont également autorisées et comptent les diapositives à partir de la fin. Par exemple, "-1" implique la dernière diapositive d'une présentation, "-2" - l'avant-dernière, etc. Comme pour les valeurs positives, lorsque le nombre de diapositives négatives dépasse le nombre total de diapositives dans la présentation donnée, il sera ajusté à la première diapositive. Le[`InsertAsNewSlide`](../insertasnewslide) La propriété booléenne est étroitement liée à celle-ci.
