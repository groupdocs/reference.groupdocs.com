## SpreadsheetSaveOptions.WorksheetNumber property

Permet d'insérer une feuille de calcul modifiée dans la copie d'une feuille de calcul existante au lieu de créer une nouvelle feuille de calcul à une seule feuille de calcul (comportement par défaut). WorksheetNumber est un nombre basé sur 1 d'une feuille de calcul dans la feuille de calcul, chargé dans la classe Editor. S'il est égal à 0 (valeur par défaut), la nouvelle feuille de calcul sera créée avec une seule feuille de calcul modifiée. S'il est supérieur ou inférieur à zéro et qu'il existe une feuille de calcul valide, chargée dans la classe Editor, la feuille de calcul modifiée, représentée par input EditableDocument, sera inséré dans cette feuille de calcul.

```csharp
public int WorksheetNumber { get; set; }
```

### Remarques

Numéro de feuille de travailLa propriété entière, si elle n'est pas dans l'état par défaut (valeur réservée '0'), représente un numéro de feuille de calcul, donc elle commence à partir de 1, pas de zéro, et sa valeur maximale est le nombre de toutes les diapositives existantes dans une présentation. Cependant, si la valeur spécifiée est supérieure à la quantité de toutes les diapositives, GroupDocs.Editor l'ajustera pour marquer la dernière feuille de calcul. Les valeurs négatives sont également autorisées et comptent les feuilles de calcul à partir de la fin. Par exemple, "-1" implique la dernière feuille de calcul dans une feuille de calcul, "-2" - l'avant-dernier, etc. Comme pour les valeurs positives, lorsque le nombre de feuilles de calcul négatif dépasse le nombre total de feuilles de calcul dans la feuille de calcul donnée, il sera ajusté à la première feuille de travail. Le[`InsertAsNewWorksheet`](../insertasnewworksheet) La propriété booléenne est étroitement liée à celle-ci.
