## WordProcessingEditOptions.InputControlsClassName property

Consente di specificare un nome di classe, che verrà inserito negli attributi 'class' in ogni elemento HTML, che rappresenta un campo nel documento di WordProcessing di input. Per impostazione predefinita è NULL - gli attributi 'class' non vengono applicati.

```csharp
public string InputControlsClassName { get; set; }
```

### Osservazioni

Quasi tutti i formati della famiglia di formati WordProcessing contengono campi, entità documento specifiche, che consentono di ottenere dati di input dagli utenti. Esiste un'ampia varietà di campi: caselle di testo, caselle di controllo, caselle combinate, elenchi a discesa, pulsanti, selettori di data/ora, ecc. Tutti sono tradotti nelle strutture e negli elementi HTML più appropriati, preservando l'utente inserito dati, se presenti nel documento di input. In casi d'uso specifici è richiesto solo di raccogliere i dati immessi sul lato client invece di modificare l'intero contenuto del documento. In tal caso è necessario identificare i controlli di input in qualche modo per recuperarli con i loro dati sul lato client. Questa proprietà consente di specificare un nome di classe, che verrà applicato per ogni controllo di input nel markup HTML, quindi il codice client sarà in grado di attraversare la struttura del documento HTML e raccogliere dati.
