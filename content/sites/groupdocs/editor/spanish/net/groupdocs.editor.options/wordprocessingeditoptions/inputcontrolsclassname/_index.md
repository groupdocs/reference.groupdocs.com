## WordProcessingEditOptions.InputControlsClassName property

Permite especificar un nombre de clase, que se colocará en los atributos de 'clase' en cada elemento HTML, que representa algún campo en el documento de procesamiento de texto de entrada. Por defecto es NULL - los atributos de 'clase' no se aplican.

```csharp
public string InputControlsClassName { get; set; }
```

### Observaciones

Casi todos los formatos de la familia de formatos de procesamiento de textos contienen campos, entidades de documentos específicas que permiten obtener datos de entrada de los usuarios. Hay una amplia variedad de campos: cuadros de texto, casillas de verificación, cuadros combinados, listas desplegables, botones, selectores de fecha/hora, etc. Todos ellos se traducen a las estructuras y elementos HTML más apropiados, conservando el usuario ingresado. datos, si están presentes en el documento de entrada. En casos de uso específicos, solo se requiere recopilar los datos ingresados en el lado del cliente en lugar de editar todo el contenido del documento. Para tal caso, se requiere identificar los controles de entrada de alguna manera para recuperarlos con sus datos en el lado del cliente. Esta propiedad permite especificar un nombre de clase, que se aplicará a cada control de entrada en el marcado HTML, por lo que el código del cliente podrá atravesar la estructura del documento HTML y recopilar datos.
