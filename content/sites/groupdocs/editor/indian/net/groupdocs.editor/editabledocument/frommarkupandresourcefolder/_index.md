## EditableDocument.FromMarkupAndResourceFolder method

स्टेटिक फ़ैक्टरी, जो एक निर्दिष्ट HTML मार्कअप और फ़ोल्डर में स्थित संसाधनों से संपादन योग्य दस्तावेज़ का एक उदाहरण बनाता है, पूर्ण पथ द्वारा निर्दिष्ट

```csharp
public static EditableDocument FromMarkupAndResourceFolder(string newHtmlContent, 
    string resourceFolderPath)
```

| पैरामीटर | प्रकार | विवरण |
| --- | --- | --- |
| newHtmlContent | String | स्ट्रिंग, जिसमें कच्चा HTML मार्कअप है, जिसे पार्स किया जाना चाहिए। खाली, खाली या अमान्य नहीं हो सकता। |
| resourceFolderPath | String | संसाधनों के साथ फ़ोल्डर के लिए अनिवार्य पथ। सभी स्टाइलशीट, जो इस फ़ोल्डर में स्थित हैं, का उपयोग किया जाएगा। NULL या रिक्त स्ट्रिंग नहीं हो सकता, और यह फ़ोल्डर मौजूद होना चाहिए। |

### प्रतिलाभ की मात्रा

संपादन योग्य दस्तावेज़ का नया गैर-शून्य उदाहरण

### टिप्पणियों

यह स्थैतिक फ़ैक्टरी तब उपयोगी होती है जब HTML दस्तावेज़ की सामग्री को एक स्ट्रिंग के रूप में प्रस्तुत किया जाता है, लेकिन सभी संसाधन किसी फ़ोल्डर में स्थित होते हैं, और अक्सर HTML मार्कअप में इन संसाधनों के लिंक अमान्य और अनुपस्थित होते हैं। इस पद्धति को लागू करते समय, यह निर्दिष्ट फ़ोल्डर को स्कैन करता है और स्वचालित रूप से दस्तावेज़ में सभी पाई गई स्टाइलशीट को लागू करता है। विभिन्न HTML संपादकों से सामग्री प्राप्त करते समय यह विधि बहुत उपयोगी होती है, जो आमतौर पर दस्तावेज़ मेटाडेटा आदि को काट देती है।
