## WordProcessingEditOptions.InputControlsClassName property

एक वर्ग नाम निर्दिष्ट करने की अनुमति देता है, जिसे प्रत्येक HTML तत्व में 'वर्ग' विशेषताओं में रखा जाएगा, जो इनपुट वर्डप्रोसेसिंग दस्तावेज़ में कुछ फ़ील्ड का प्रतिनिधित्व करता है। डिफ़ॉल्ट रूप से NULL है - 'वर्ग' विशेषताएँ लागू नहीं होती हैं।

```csharp
public string InputControlsClassName { get; set; }
```

### टिप्पणियों

WordProcessing स्वरूप परिवार के लगभग सभी स्वरूपों में फ़ील्ड होते हैं — विशिष्ट दस्तावेज़ निकाय, जो उपयोगकर्ताओं से इनपुट डेटा प्राप्त करने की अनुमति देते हैं। फ़ील्ड्स की व्यापक विविधताएँ हैं: टेक्स्ट-बॉक्सेज़, चेकबॉक्सेज़, कॉम्बो-बॉक्सेज़, ड्रॉप डाउन सूचियाँ, बटन, दिनांक/समय पिकर, आदि। इन सभी को सबसे उपयुक्त HTML संरचनाओं और तत्वों में अनुवादित किया जाता है, जिसमें दर्ज किए गए उपयोगकर्ता को संरक्षित किया जाता है। डेटा, अगर वे इनपुट दस्तावेज़ में मौजूद हैं। विशिष्ट उपयोग-मामलों में संपूर्ण दस्तावेज़ सामग्री को संपादित करने के बजाय क्लाइंट-साइड पर केवल दर्ज किए गए डेटा को इकट्ठा करना आवश्यक है। ऐसे मामले के लिए क्लाइंट-साइड पर अपने डेटा के साथ उन्हें लाने के लिए किसी तरह से इनपुट नियंत्रणों की पहचान करना आवश्यक है। यह संपत्ति एक वर्ग नाम निर्दिष्ट करने की अनुमति देती है, जिसे HTML मार्कअप में प्रत्येक इनपुट नियंत्रण के लिए लागू किया जाएगा, इसलिए क्लाइंट कोड HTML दस्तावेज़ संरचना को पार करने और डेटा एकत्र करने में सक्षम होगा।
