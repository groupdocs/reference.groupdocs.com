## MhtmlSaveOptions.ExportCidUrls property

Menentukan apakah akan menggunakan URL CID (Content-ID) untuk mereferensikan sumber daya (gambar, font, CSS) yang disertakan dalam dokumen MHTML. Nilai default adalah`PALSU` .

```csharp
public bool ExportCidUrls { get; set; }
```

### Perkataan

Secara default, sumber daya dalam dokumen MHTML direferensikan berdasarkan nama file (misalnya, "image.png"), yang dicocokkan dengan header "Content-Location" bagian MIME. Opsi ini mengaktifkan metode alternatif, di mana referensi ke file sumber ditulis sebagai URL CID (Content-ID) (misalnya, "cid:image.png") dan dicocokkan dengan header "Content-ID".

Secara teori, seharusnya tidak ada perbedaan antara kedua metode referensi dan keduanya harus bekerja dengan baik di browser atau agen email mana pun. Namun, dalam praktiknya, beberapa agen gagal mengambil sumber daya dengan nama file. Jika browser atau agen email Anda menolak memuat sumber daya yang disertakan dalam dokumen MTHML (tidak menampilkan gambar atau tidak memuat gaya CSS), coba ekspor dokumen dengan URL CID.
