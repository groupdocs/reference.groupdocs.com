## Editor.GetDocumentInfo method

Mengembalikan metadata tentang dokumen, yang dimuat ke instance 'Editor' ini

```csharp
public IDocumentInfo GetDocumentInfo(string password)
```

| Parameter | Jenis | Keterangan |
| --- | --- | --- |
| password | String | Pengguna dapat menentukan kata sandi untuk dokumen, jika dokumen ini dienkripsi dengan kata sandi. Mungkin NULL atau string kosong, yang setara dengan kata sandi yang tidak ada. Untuk format dokumen tersebut, yang tidak memiliki fitur proteksi kata sandi, argumen ini akan diabaikan. Jika dokumen dienkripsi, dan kata sandi tidak ditentukan dalam parameter ini, tetapi ditentukan sebelumnya dalam opsi muat saat membuat ini[`Editor`](../../editor) misalnya, itu akan digunakan. |

### Nilai Pengembalian

Pewaris khusus format dari[`IDocumentInfo`](../../../groupdocs.editor.metadata/idocumentinfo) antarmuka, yang menunjukkan format yang terdeteksi dengan metadata khusus format, atau NULL, jika dokumen tidak dikenali sebagai dapat didukung atau rusak.

### Pengecualian

| pengecualian | kondisi |
| --- | --- |
| ObjectDisposedException | Dilempar saat instance Editor sudah dibuang saat "GetDocumentInfo" dipanggil |
| [PasswordRequiredException](../../passwordrequiredexception) | Dilempar saat memuat dokumen dilindungi kata sandi, tetapi kata sandi tidak ditentukan dalam parameter "*password*" |
| [IncorrectPasswordException](../../incorrectpasswordexception) | Dilempar ketika dokumen yang dimuat dilindungi kata sandi, kata sandi ditentukan, tetapi salah |
| InvalidOperationException | Dilempar ketika kesalahan tak terduga yang sifatnya tidak diketahui telah terjadi |

### Perkataan

Metode GetDocumentInfo berguna ketika tidak jelas format mana yang merupakan dokumen masukan, apakah dilindungi kata sandi dan/atau berapa banyak halaman/lembar kerja/slide yang ada di dalamnya. Berdasarkan metadata ini, dikembalikan oleh GetDocumentInfo, dimungkinkan untuk menyesuaikan dengan benar opsi muat dan edit untuk saluran pemrosesan utama.

Metode GetDocumentInfo selalu mengembalikan data lengkap, tidak terpengaruh oleh mode uji coba, penggunaannya tidak menghapus byte atau kredit yang dikonsumsi.
