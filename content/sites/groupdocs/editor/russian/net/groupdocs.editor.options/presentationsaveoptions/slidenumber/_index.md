## PresentationSaveOptions.SlideNumber property

Позволяет вставить отредактированный слайд в существующую презентацию вместо создания новой презентации, состоящей из одного слайда (поведение по умолчанию). Номер слайда — это номер слайда в презентации, начинающийся с 1 и загруженный в класс Editor. Если он равен 0 (значение по умолчанию), новая презентация будет создана с одним отредактированным слайдом. Если оно больше или меньше нуля, и в класс Editor загружена допустимая презентация, отредактированный слайд, хранящийся во входном экземпляре EditableDocument, будет вставлен в эту презентацию.

public int SlideNumber { get; set; }

### Примечания

номер слайдацелочисленное свойство, если оно не находится в состоянии по умолчанию (зарезервированное значение «0»), представляет номер слайда, поэтому оно начинается с 1, а не с нуля, а его максимальное значение — это количество всех существующих слайдов в презентации. Однако, если указанное значение больше, чем количество всех слайдов, GroupDocs.Editor отрегулирует его, чтобы отметить последний слайд. Также допускаются отрицательные значения и подсчет слайдов с конца. Например, «-1» означает последний слайд в презентации, «-2» — предпоследний и т. д. Как и в случае с положительными значениями, когда отрицательное число слайдов превышает общее количество слайдов в данной презентации, оно будет изменено на первый слайд. [`InsertAsNewSlide`](../insertasnewslide) логическое свойство тесно связано с этим.
