## SpreadsheetSaveOptions.WorksheetNumber property

Позволяет вставить отредактированный рабочий лист в копию существующей электронной таблицы вместо создания новой электронной таблицы с одним рабочим листом (поведение по умолчанию). WorksheetNumber — это номер рабочего листа в электронной таблице, начинающийся с 1, загруженный в класс Editor. Если он равен 0 (значение по умолчанию), новая электронная таблица будет создана с одним редактируемым листом. входной экземпляр EditableDocument, будет вставлен в эту электронную таблицу.

```csharp
public int WorksheetNumber { get; set; }
```

### Примечания

номер рабочего листацелочисленное свойство, если оно не находится в состоянии по умолчанию (зарезервированное значение «0»), представляет номер рабочего листа, поэтому оно начинается с 1, а не с нуля, а его максимальное значение — это количество всех существующих слайдов в презентации. Однако, если указанное значение больше, чем количество всех слайдов, GroupDocs.Editor отрегулирует его, чтобы отметить последний рабочий лист. Отрицательные значения также допускаются и учитывают рабочие листы с конца. Например, «-1» означает последний рабочий лист в электронной таблице, «-2» — предпоследний и т. д. Как и в случае с положительными значениями, когда отрицательное число рабочих листов превышает общее количество рабочих листов в данной электронной таблице, оно будет изменено на первый рабочий лист. [`InsertAsNewWorksheet`](../insertasnewworksheet) логическое свойство тесно связано с этим.
