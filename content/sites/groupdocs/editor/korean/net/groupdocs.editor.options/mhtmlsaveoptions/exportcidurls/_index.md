## MhtmlSaveOptions.ExportCidUrls property

MHTML 문서에 포함된 리소스(이미지, 글꼴, CSS)를 참조하기 위해 CID(Content-ID) URL을 사용할지 여부를 지정합니다. 기본값은`거짓` .

```csharp
public bool ExportCidUrls { get; set; }
```

### 비고

기본적으로 MHTML 문서의 리소스는 MIME 부분의 "Content-Location" 헤더와 일치하는 파일 이름(예: "image.png")으로 참조됩니다. 이 옵션은 리소스 파일에 대한 참조가 CID(Content-ID) URL(예: "cid:image.png")로 작성되고 "Content-ID" 헤더와 일치하는 대체 방법을 활성화합니다.

이론적으로 두 참조 방법 간에 차이가 없어야 하며 둘 중 하나가 모든 브라우저나 메일 에이전트에서 제대로 작동해야 합니다. 그러나 실제로 일부 에이전트는 파일 이름으로 리소스를 가져오지 못합니다. 브라우저 또는 메일 에이전트가 MTHML 문서에 포함된 리소스 로드를 거부하는 경우(이미지를 표시하지 않거나 CSS 스타일을 로드하지 않음) CID URL로 문서를 내보내십시오.
