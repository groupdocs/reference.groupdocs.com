## PresentationSaveOptions.SlideNumber property

새 단일 슬라이드 프레젠테이션을 만드는 대신 기존 프레젠테이션에 편집된 슬라이드를 삽입할 수 있습니다(기본 동작). 슬라이드 번호는 Editor 클래스에 로드된 프레젠테이션의 슬라이드 번호로 1부터 시작합니다. 0(기본값)이면 단일 편집 슬라이드로 새 프레젠테이션이 생성됩니다. 0보다 크거나 작으며 Editor 클래스에 로드된 유효한 프레젠테이션이 있는 경우 입력 EditableDocument 인스턴스 내에 저장된 편집된 슬라이드가 이 프레젠테이션에 삽입됩니다.

```csharp
public int SlideNumber { get; set; }
```

### 비고

슬라이드 번호정수 속성은 기본 상태(예약값 '0')가 아닌 경우 슬라이드 번호를 나타내므로 0이 아닌 1부터 시작하며 최대값은 프레젠테이션에 존재하는 모든 슬라이드의 양입니다. 그러나 지정된 값이 모든 슬라이드의 양보다 큰 경우 GroupDocs.Editor는 마지막 슬라이드를 표시하도록 조정합니다. 음수 값도 허용되며 끝에서부터 슬라이드를 계산합니다. 예를 들어 "-1"은 프리젠테이션의 마지막 슬라이드를 의미하고 "-2"는 마지막 슬라이드를 의미합니다. 양수 값과 마찬가지로 음수 슬라이드 번호가 주어진 프레젠테이션의 총 슬라이드 수를 초과하면 다음과 같이 조정됩니다. 첫 번째 슬라이드. [`InsertAsNewSlide`](../insertasnewslide) 부울 속성은 이것과 밀접하게 결합되어 있습니다.

### 예

주어진 프레젠테이션에는 5개의 슬라이드가 있습니다. SlideNumber = 0; — 주어진 프리젠테이션을 무시하고 새로운 프리젠테이션을 생성하고 편집된 슬라이드를 여기에 넣습니다. SlideNumber = 1; — 첫 번째 슬라이드를edited SlideNumber = 2로 바꿉니다. — 두 번째 슬라이드를 edit SlideNumber = 5로 바꿉니다. — 마지막(5번째) 슬라이드를edited SlideNumber = 6으로 교체합니다. — 마지막(5번째) 슬라이드를 편집된 슬라이드로 바꿉니다. 6은 5보다 크고 조정 SlideNumber = -1이기 때문입니다. — "-1"은 "마지막 기존"을 의미하기 때문에 마지막(5번째) 슬라이드를 편집된 것으로 바꿉니다. SlideNumber = -2; — 네 번째 슬라이드를edited SlideNumber = -3으로 바꿉니다. — 3번째 슬라이드를edited SlideNumber = -4로 바꿉니다. — 두 번째 슬라이드를edited SlideNumber = -5로 바꿉니다. — 첫 번째 슬라이드를edited SlideNumber = -6으로 바꿉니다. — "-6"이 5보다 크므로 조정된 이므로 첫 번째 슬라이드를 편집된 것으로 바꿉니다.
