## MhtmlSaveOptions.ExportCidUrls property

Καθορίζει εάν θα χρησιμοποιηθούν διευθύνσεις URL CID (Content-ID) για αναφορά πόρων (εικόνες, γραμματοσειρές, CSS) που περιλαμβάνονται σε έγγραφα MHTML. Η προεπιλεγμένη τιμή είναι`ψευδής` .

```csharp
public bool ExportCidUrls { get; set; }
```

### Παρατηρήσεις

Από προεπιλογή, οι πόροι στα έγγραφα MHTML αναφέρονται με όνομα αρχείου (για παράδειγμα, "image.png"), τα οποία αντιστοιχίζονται με τις κεφαλίδες "Content-Location" των τμημάτων MIME. Αυτή η επιλογή ενεργοποιεί μια εναλλακτική μέθοδο, όπου οι αναφορές σε αρχεία πόρων γράφονται ως διευθύνσεις URL CID (Content-ID) (για παράδειγμα, "cid:image.png") και αντιστοιχίζονται με τις κεφαλίδες "Content-ID".

Θεωρητικά, δεν θα πρέπει να υπάρχει διαφορά μεταξύ των δύο μεθόδων αναφοράς και καμία από αυτές θα πρέπει να λειτουργεί καλά σε οποιοδήποτε πρόγραμμα περιήγησης ή πράκτορα αλληλογραφίας. Στην πράξη, ωστόσο, ορισμένοι πράκτορες αποτυγχάνουν να ανακτήσουν πόρους με βάση το όνομα αρχείου. Εάν το πρόγραμμα περιήγησής σας ή ο αντιπρόσωπος αλληλογραφίας αρνείται να φορτώσει τους πόρους που περιλαμβάνονται σε ένα έγγραφο MTHML (δεν εμφανίζει εικόνες ή δεν φορτώνει στυλ CSS), δοκιμάστε να εξαγάγετε το έγγραφο με διευθύνσεις URL CID.
