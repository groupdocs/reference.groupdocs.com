Der Namespace bietet Funktionen, die für die Arbeit mit digitalen Signaturen auf der Grundlage der Syntax von PKCS #7, spezifiziert in RFC 5652. , vorgesehen sind.

## Klassen

| Klasse | Beschreibung |
| --- | --- |
| [Cms](./cms) | Stellt ein digitales Zeichen dar, das mit Cryptographic Message Syntax (CMS) erstellt wurde – dem IETF-Standard für kryptografisch geschützte Nachrichten. CMS basiert auf der Syntax von PKCS #7, spezifiziert in RFC 5652. Siehe bitte[https://tools.ietf.org/html/rfc5652](https://tools.ietf.org/html/rfc5652) für weitere Informationen. |
| [CmsAttribute](./cmsattribute) | Stellt ein CMS-Unterzeichner-Info-Attribut dar. |
| [CmsCertificate](./cmscertificate) | Stellt ein CMS-Zertifikat dar. |
| [CmsEncapsulatedContent](./cmsencapsulatedcontent) | Stellt einen signierten Inhaltscontainer dar, der aus einer Inhaltstypkennung und dem Inhalt selbst besteht. |
| [CmsPackage](./cmspackage) | Stellt ein CMS-Metadatenpaket für digitale Signaturen dar. |
| [CmsSigner](./cmssigner) | Stellt CMS-Informationen pro Unterzeichner dar. |
| [Oid](./oid) | Stellt eine Objektkennung (OID) dar. Ein von der International Telecommunications Union (ITU) und ISO/IEC standardisierter Identifikatormechanismus zur Benennung von Objekten, Konzepten oder "Dingen" mit einem global eindeutigen dauerhaften Namen. |
