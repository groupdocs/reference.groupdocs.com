名前空間は、RFC 5652 で指定されている PKCS #7 の構文に基づいてデジタル署名を操作するための機能を提供します。

## クラス

| クラス | 説明 |
| --- | --- |
| [Cms](./cms) | は、Cryptographic Message Syntax (CMS) (暗号で保護されたメッセージに対する IETF の標準) で作成されたデジタル署名を表します。 CMS は、RFC 5652 で指定されている PKCS #7 の構文に基づいています。 を参照してください。[https://tools.ietf.org/html/rfc5652](https://tools.ietf.org/html/rfc5652)詳しくは. |
| [CmsAttribute](./cmsattribute) | CMS 署名者情報属性を表します。 |
| [CmsCertificate](./cmscertificate) | CMS 証明書を表します。 |
| [CmsEncapsulatedContent](./cmsencapsulatedcontent) | コンテンツ タイプ識別子とコンテンツ自体で構成される、署名されたコンテンツ コンテナーを表します。 |
| [CmsPackage](./cmspackage) | CMS デジタル署名メタデータ パッケージを表します。 |
| [CmsSigner](./cmssigner) | 署名者ごとの CMS 情報を表します。 |
| [Oid](./oid) | オブジェクト識別子 (OID) を表します。 国際電気通信連合 (ITU) および ISO/IEC によって標準化された、オブジェクト、概念、または「もの」にグローバルに明確で永続的な名前を付けるための識別子メカニズム。 |
