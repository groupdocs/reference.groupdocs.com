L'espace de noms fournit des fonctionnalités destinées à fonctionner avec des signatures numériques basées sur la syntaxe de PKCS #7, spécifiée dans la RFC 5652.

## Des classes

| Classer | La description |
| --- | --- |
| [Cms](./cms) | Représente un signe numérique créé avec la syntaxe de message cryptographique (CMS) - la norme de l'IETF pour les messages protégés par chiffrement. CMS est basé sur la syntaxe de PKCS #7, spécifiée dans la RFC 5652. Veuillez consulter[https://tools.ietf.org/html/rfc5652](https://tools.ietf.org/html/rfc5652) pour plus d'informations. |
| [CmsAttribute](./cmsattribute) | Représente un attribut d'informations de signataire CMS. |
| [CmsCertificate](./cmscertificate) | Représente un certificat CMS. |
| [CmsEncapsulatedContent](./cmsencapsulatedcontent) | Représente un conteneur de contenu signé, composé d'un identifiant de type de contenu et du contenu lui-même. |
| [CmsPackage](./cmspackage) | Représente un package de métadonnées de signature numérique CMS. |
| [CmsSigner](./cmssigner) | Représente les informations CMS par signataire. |
| [Oid](./oid) | Représente un identificateur d'objet (OID). Un mécanisme d'identification normalisé par l'Union internationale des télécommunications (UIT) et l'ISO/CEI pour nommer tout objet, concept ou « chose » avec un nom persistant globalement non ambigu. |
