### Perkataan

Objek sumber data dapat berupa salah satu dari jenis berikut:

* [`XmlDataSource`](../../../groupdocs.assembly.data/xmldatasource)
* [`JsonDataSource`](../../../groupdocs.assembly.data/jsondatasource)
* [`CsvDataSource`](../../../groupdocs.assembly.data/csvdatasource)
* [`DocumentTableSet`](../../../groupdocs.assembly.data/documenttableset)
* [`DocumentTable`](../../../groupdocs.assembly.data/documenttable)
* DataSet
* DataTable
* DataRow
* IDataReader
* IDataRecord
* DataView
* DataRowView
* Tipe .NET non-dinamis dan non-anonim arbitrer lainnya

Untuk informasi tentang cara bekerja dengan sumber data dari tipe yang berbeda dalam dokumen template, lihat referensi sintaks template (https://docs.groupdocs.com/display/assemblynet/Template+Syntax+-+Part+1+of+2# TemplateSyntax-Part1of2-UsingDataSources).

Ketika nama objek sumber data ditentukan, Anda dapat mengakses objek sumber data dan anggotanya dalam dokumen template menggunakan nama.

Ketika nama objek sumber data null atau kosong, Anda masih dapat mengakses anggota objek sumber data dalam dokumen template menggunakan akses anggota objek konteks (lihat Referensi Sintaks Template untuk informasi lebih lanjut), tetapi Anda tidak dapat mengakses objek sumber data itu sendiri.

Saat melewati beberapa[`DataSourceInfo`](../../datasourceinfo) contoh ke[`DocumentAssembler`](../../documentassembler) , hanya nama objek sumber data pertama yang boleh nol atau kosong. Nama yang lainnya harus ditentukan dan unik.
