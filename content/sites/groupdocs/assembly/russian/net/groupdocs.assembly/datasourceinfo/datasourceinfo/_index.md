## DataSourceInfo(object, string) {#constructor_2}

Создает новый экземпляр этого класса с объектом источника данных и указанным именем.

```csharp
public DataSourceInfo(object dataSource, string name)
```

| Параметр | Тип | Описание |
| --- | --- | --- |
| dataSource | Object | Объект источника данных. |
| name | String | Имя объекта источника данных, который будет использоваться для доступа к объекту источника данных в шаблоне документа. |

### Примечания

Объект источника данных может быть одного из следующих типов:

* [`XmlDataSource`](../../../groupdocs.assembly.data/xmldatasource)
* [`JsonDataSource`](../../../groupdocs.assembly.data/jsondatasource)
* [`CsvDataSource`](../../../groupdocs.assembly.data/csvdatasource)
* [`DocumentTableSet`](../../../groupdocs.assembly.data/documenttableset)
* [`DocumentTable`](../../../groupdocs.assembly.data/documenttable)
* DataSet
* DataTable
* DataRow
* IDataReader
* IDataRecord
* DataView
* DataRowView
* Любой другой произвольный нединамический и неанонимный тип .NET

Для получения информации о том, как работать с источниками данных разных типов в документах-шаблонах, см. справочник по синтаксису шаблона (https://docs.groupdocs.com/display/assemblynet/Template+Syntax+-+Part+1+of+2# TemplateSyntax-Part1of2-UsingDataSources).

Когда имя объекта источника данных указано, вы можете получить доступ к объекту источника данных и его членам в шаблоне документа, используя имя.

Когда имя объекта источника данных равно null или пусто, вы все равно можете получить доступ к членам объекта источника данных в документе шаблона, используя доступ к членам объекта контекста (дополнительную информацию см. в Справочнике по синтаксису шаблона), , но вы не можете получить доступ к сам объект источника данных.

При передаче нескольких[`DataSourceInfo`](../../datasourceinfo) случаи[`DocumentAssembler`](../../documentassembler) , только имя первого объекта источника данных может быть нулевым или пустым. Имена остальных должны быть указаны и уникальны.
