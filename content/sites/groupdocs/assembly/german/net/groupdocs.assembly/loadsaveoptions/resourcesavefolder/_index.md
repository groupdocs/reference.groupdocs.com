## LoadSaveOptions.ResourceSaveFolder property

Ruft einen Pfad zu einem Ordner ab oder legt ihn fest, um externe Ressourcendateien zu speichern, während ein zusammengestelltes Dokument, das aus einem Nicht-HTML- -Format geladen wurde, in HTML gespeichert wird. Der Standardwert ist eine leere Zeichenfolge.

```csharp
public string ResourceSaveFolder { get; set; }
```

### Bemerkungen

Beim Speichern eines zusammengestellten Dokuments in einer HTML-Datei werden externe Ressourcendateien standardmäßig in einem Ordner gespeichert, der denselben Namen wie die HTML-Datei ohne Erweiterung plus das Suffix „_files“ hat. Dieser Ordner befindet sich in demselben Ordner wie die HTML-Datei. Dies ist jedoch nicht möglich, wenn ein zusammengesetztes Dokument in einem HTML-Stream gespeichert wird. Legen Sie diese Eigenschaft fest, um einen Pfad zu einem Ordner anzugeben, in dem externe Ressourcendateien gespeichert werden, wenn ein zusammengesetztes Dokument in einem HTML-Stream gespeichert wird, oder um den Standardordner beim Speichern zu überschreiben zusammengesetztes Dokument in eine HTML-Datei.

Ein Wert dieser Eigenschaft wird ignoriert, wenn ein zusammengestelltes Dokument, das in HTML gespeichert wird, auch von HTML geladen wurde (externe Ressourcendateien werden nicht gespeichert und Links zu ihnen werden dann nicht geändert).
