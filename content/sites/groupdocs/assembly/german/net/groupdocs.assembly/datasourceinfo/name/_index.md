## DataSourceInfo.Name property

Ruft den Namen des Datenquellenobjekts ab oder legt ihn fest, das für den Zugriff auf das Datenquellenobjekt in einem Vorlagendokument verwendet werden soll.

```csharp
public string Name { get; set; }
```

### Bemerkungen

Wenn der Name des Datenquellenobjekts angegeben ist, können Sie auf das Datenquellenobjekt und seine Mitglieder in einem Vorlagendokument zugreifen, indem Sie den Namen verwenden.

Wenn der Name des Datenquellenobjekts null oder leer ist, können Sie immer noch auf Mitglieder des Datenquellenobjekts in einem Vorlagendokument zugreifen, indem Sie Kontextobjekt-Mitgliederzugriff verwenden (weitere Informationen finden Sie unter Vorlagensyntaxreferenz), , aber Sie können nicht auf die zugreifen Datenquellenobjekt selbst.

Beim Überholen mehrerer[`DataSourceInfo`](../../datasourceinfo) Instanzen zu[`DocumentAssembler`](../../documentassembler) , nur der Name von des ersten Datenquellenobjekts kann null oder leer sein. Die Namen der übrigen müssen angegeben und eindeutig sein.
