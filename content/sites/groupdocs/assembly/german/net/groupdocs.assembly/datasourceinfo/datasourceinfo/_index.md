## DataSourceInfo(object, string) {#constructor_2}


| Parameter | Typ | Beschreibung |
| --- | --- | --- |
| dataSource | Object | Das Datenquellenobjekt. |
| name | String | Der Name des Datenquellenobjekts, das für den Zugriff auf das Datenquellenobjekt in einem Vorlagendokument verwendet werden soll. |

### Bemerkungen

Das Datenquellenobjekt kann einen der folgenden Typen haben:

* [`XmlDataSource`](../../../groupdocs.assembly.data/xmldatasource)
* [`JsonDataSource`](../../../groupdocs.assembly.data/jsondatasource)
* [`CsvDataSource`](../../../groupdocs.assembly.data/csvdatasource)
* [`DocumentTableSet`](../../../groupdocs.assembly.data/documenttableset)
* [`DocumentTable`](../../../groupdocs.assembly.data/documenttable)
* DataSet
* DataTable
* DataRow
* IDataReader
* IDataRecord
* DataView
* DataRowView
* Jeder andere beliebige nicht dynamische und nicht anonyme .NET-Typ

Informationen zum Arbeiten mit Datenquellen verschiedener Typen in Vorlagendokumenten finden Sie in der Vorlagensyntaxreferenz (https://docs.groupdocs.com/display/assemblynet/Template+Syntax+-+Part+1+of+2# TemplateSyntax-Teil1von2-UsingDataSources).

Wenn der Name des Datenquellenobjekts angegeben ist, können Sie auf das Datenquellenobjekt und seine Mitglieder in einem Vorlagendokument zugreifen, indem Sie den Namen verwenden.

Wenn der Name des Datenquellenobjekts null oder leer ist, können Sie immer noch auf Mitglieder des Datenquellenobjekts in einem Vorlagendokument zugreifen, indem Sie Kontextobjekt-Mitgliederzugriff verwenden (weitere Informationen finden Sie unter Vorlagensyntaxreferenz), , aber Sie können nicht auf die zugreifen Datenquellenobjekt selbst.

Beim Überholen mehrerer[`DataSourceInfo`](../../datasourceinfo) Instanzen zu[`DocumentAssembler`](../../documentassembler) , nur der Name von des ersten Datenquellenobjekts kann null oder leer sein. Die Namen der übrigen müssen angegeben und eindeutig sein.
